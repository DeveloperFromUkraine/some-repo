Adding to the Doc Site

  • If you would like to use the automated documentation generator to add to the doc site, run the following from the terminal:
    • npm install --registry=http://artifactory.mia.ulti.io/artifactory/api/npm/npm generator-ign
  • After installing the package, run the following from the terminal:
    • npm install -g yo
  • After installing yeoman, run the following from the terminal in the directory where the generator is installed:
    • yo ign
  • Be sure to separate all words within the name of your component with hyphens, i.e. "demo-app-example", when prompted.
  • This package allows for semi-automatic generation of component documentation. Just check the output and push to the repository!
  • You can use this feature for out of the box angular material components and subcomponents or custom components and subcomponents.

OR
  • Below are the requirements to add to the doc site, if you choose to do so manually. Please read these instructions thoroughly and mind our restrictions!

How to Add Your Own Component:

HTML Requirements:

  • A Use description.
    • This informs the consumer how to use the component and should provide any contextual links.
  • A Sample.
    • The doc site samples are embedded within a tabs module, which can be found here. Samples should include an implementation, the HTML for the component, and the typescript if necessary.
  • A ReadMe.
    • This will provide the content and style descriptions.

ReadMe Requirements:

  • Follow the Markdown component styling.
  • A Content Description
    • This provides the consumer with information on what to include within the component.
  • A Style Description
    • This provides the consumer with information on how to style the component. If the component is a container or does not have stylistic aspects, this should be stated outright.
  • You may add other information about your component here! Some components require or are enhanced with API information, for example. It's your component! Do your best to make it as consumable as possible!

Troubleshooting

  • "npm ERR! enoent ENOENT: no such file or directory, access error" when running npm install on the generator.
    • Run rm -rf node_modules and attempt the install again.
  • "You don't seem to have a generator with the name “ign” installed" error when running the yo ign command.
    • Run "npm link" from the terminal in the generator's directory.
    • Run "npm link generator-ign" from the terminal in the generator's directory, if the above fails to work.
  • "This usually happens because your environment has changed since running npm install" error when running yarn start from the terminal.
    • Run rm -rf node_module && yarn install.
  • "Cannot find command 'yo'" error when running yo ign from the terminal.
    • Run npm install -g yo from the terminal.
