Hello! You've stumbled upon the NU team's doc site.

This is where we store our components that can be universally utilized within the web app. This helps to improve consistency, writability, and readability in code across the board!
Click on the hamburger menu icon to begin navigating to our components. You can consume these components and use them in your own development!



Adding to the Doc Site


Below are the requirements to add to the doc site. Please read these instructions thoroughly and mind our restrictions!

How to Add Your Own Component:



HTML Requirements:


  • A Use description.
    • This informs the consumer how to use the component and should provide any contextual links.
  • A Sample.
    • The doc site samples are embedded within a tabs module, which can be found here. Samples should include an implementation, the HTML for the component, and the typescript if necessary.
  • A ReadMe.
    • This will provide the content and style descriptions.

ReadMe Requirements:


  • A Content Description
    • This provides the consumer with information on what to include within the component.
  • A Style Description
    • This provides the consumer with information on how to style the component. If the component is a container or does not have stylistic aspects, this should be stated outright.
  • You may add other information about your component here! Some components require or are enhanced with API information, for example. It's your component! Do your best to make it as consumable as possible!
