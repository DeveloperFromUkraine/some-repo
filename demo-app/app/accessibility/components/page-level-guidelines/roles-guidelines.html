Roles

The following are page level, roles guidelines should be adhered to when implementing new pages.

  • Why?

    • Roles help to improve accessibility by describing the structure of and providing keyboard navigation functionality for webpages.

  • Where?

    • An aria role or role attribute should be added inline to the opening tag of the element.

    Common instances in which roles should be implemented:

    • When implementing icons as button, or with button functionality.
    • Navigational elements, such as side navs, and lists where multiple elements exist in succession.
    • Banners, such as information and error banners, that may not initially be established on the page.
    • When there is content allocated on the page that can only be accessed after navigational elements.

  • Code Examples

    • List Items
      • 
                    <ign-list-item role="listitem"></ign-list-item>
    • Main Content
      • 
                    <div id="body" role="main"></div>
    • Icons
      • 
                    <mat-icon role="button"></mat-icon>
    • Banners
      • 
                    <ign-error-banner role="banner"></ign-error-banner>
